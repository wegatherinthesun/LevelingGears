# pipeline/ — Leveling Gears data pipeline

Not part of the addon itself — this is offline tooling that produces the `Items`/`Sources`/`Quests`/
`Chains`/`Recipes`/`BySlot` Lua files `ROADMAP.md`'s `0.4` step calls for. See
[`../DATA_PIPELINE.md`](../DATA_PIPELINE.md) for the full source/schema/parser design and known gaps.

## Current state

`big_data.py --build-database` runs the real extraction end-to-end against cmangos tbc-db's current
SQL dump and writes `Items.lua`/`Sources.lua`/`Quests.lua`/`Chains.lua`/`Recipes.lua`/`BySlot.lua` to
`output/` — real data (a recent run: 18,711 Items, 6,599 Quests, 1,108 Chains, 900 Recipes), not a
sample. Not wired into the addon itself yet — this is staging output for review.

Questie is not included (its source license is unresolved — see `DATA_PIPELINE.md`). Known gaps in
what's extracted so far (also in `DATA_PIPELINE.md`'s Status note and `ROADMAP.md`'s `0.41-0.44`/
`0.46` entries): recipe reagents/created-items are empty (`spell_template` ships empty in this
dump), `zone` is a numeric map id not a real zone name, and shared loot-pool items are collapsed to
one representative source each (a stopgap pending `ROADMAP.md`'s `0.46` data-curation phase).

## Running it

```
python3 big_data.py --build-database
```

Flags:
- `--build-database` — run every extractor (Items/Sources/Quests/Chains/Recipes) and write the full
  `output/*.lua` set, plus `BySlot`'s index.
- `--extract-recipes` — run just the recipe extractor (superseded by `--build-database`, kept for
  quick iteration on that extractor alone).
- `--skip-download` — reuse the `.sql.gz` already in `downloads/` instead of fetching it again.
- `--verbose` — DEBUG-level logging.

With no flags, it only downloads the dump and writes `output/schema_report.txt` (confirms real
table/column names — useful before touching an extractor, not needed for a normal run).

No third-party packages required — standard library only.

## Files

- `dependencies.py` — checks Python version + disk space, prints install steps for anything missing.
- `download.py` — fetches cmangos's current `Full_DB/*.sql.gz` over plain HTTPS (no `git` needed).
- `inspect_schema.py` — confirms real `CREATE TABLE` names/columns, writes `schema_report.txt`.
- `sql_extract.py` — hand-rolled mysqldump `INSERT` row streamer, reused by every extractor.
- `lua_writer.py` — serializes Python data into Lua table literal source.
- `wow_enums.py` — stable WoW client constants (InventoryType/ItemClass/race bitmask).
- `extract_items.py` / `extract_loot.py` / `extract_vendor.py` / `extract_quests.py` /
  `extract_recipes.py` — one extractor per schema table.
- `build_database.py` — runs every extractor, merges Sources, builds `BySlot`, writes all six
  `output/*.lua` files.

## Where things land

- `downloads/` — the raw `.sql.gz` and decompressed `.sql` (gitignored, ~390 MB+, re-downloadable).
- `output/` — `schema_report.txt` plus the six generated `*.lua` files (gitignored, regenerated by
  every run).
- `logs/` — one timestamped log file per run, mirroring everything printed to the console
  (gitignored).
